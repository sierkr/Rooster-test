Rooster v3.27.105 — Release notes
==================================

EXPORT/IMPORT BESTAND-BESTENDIG (STABIEL STOEL-ID)
-----------------------------
De Excel-koppeling is nu wissel-bestendig.

- Export legt een verborgen blad "_kolommen" vast: welke code bij welke stoel
  hoort op het moment van export. Het zichtbare blad blijft gelijk.
- Import gebruikt die stabiele stoel-id; valt terug op de code voor oudere
  bestanden. Een oude kolom (bv. 'BL') landt zo na een wissel nog steeds op de
  juiste stoel — geen stil dataverlies meer.
- Niet-koppelbare of ontbrekende kolommen worden nu als WAARSCHUWING getoond in
  het importrapport, in plaats van stil overgeslagen.

Oudere Excel-bestanden (zonder verborgen blad) blijven werken via de
code-fallback, nu mét waarschuwing waar nodig.
